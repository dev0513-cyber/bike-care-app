# BikeCare Deployment Guide - Free Hosting

## Option 1: Railway.app (Recommended - Free Tier)

### Backend Setup (Django)

1. **Push to GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/bikecare.git
   git push -u origin main
   ```

2. **Create Railway Account**
   - Go to https://railway.app
   - Sign up with GitHub
   - Connect your repository

3. **Create Backend Service**
   - New Project → GitHub Repo → bikecare
   - Click "Add Variables"
   - Set environment variables:
     ```
     DJANGO_SETTINGS_MODULE=bikecare.settings_prod
     SECRET_KEY=your-secret-key-here
     DEBUG=False
     ALLOWED_HOSTS=yourdomain.railway.app,localhost
     DATABASE_URL=postgresql://... (auto-generated by Railway)
     ```

4. **Set up build command in Railway**
   - Root Directory: `backend`
   - Build Command: `pip install -r requirements.txt && python manage.py migrate && python manage.py collectstatic --noinput`
   - Start Command: `gunicorn bikecare.wsgi:application --bind 0.0.0.0:$PORT`

5. **Deploy Frontend on Vercel**
   - Go to https://vercel.com
   - Import GitHub repo
   - Framework: Next.js (or Create React App)
   - Update API URL in frontend to point to Railway backend

## Option 2: Render.com (Free - Sleeps after 15min)

1. **Push to GitHub** (same as above)

2. **Create Render Account**
   - Go to https://render.com
   - Sign up with GitHub

3. **Create Web Service**
   - New → Web Service → Connect GitHub repo
   - Name: bikecare-backend
   - Environment: Python 3.11
   - Build Command:
     ```
     pip install -r requirements.txt && python manage.py migrate && python manage.py collectstatic --noinput
     ```
   - Start Command:
     ```
     gunicorn bikecare.wsgi:application --bind 0.0.0.0:$PORT
     ```
   - Add environment variables (same as Railway)

4. **Database Setup**
   - Render provides free PostgreSQL
   - Or use Railway's free PostgreSQL

## Option 3: PythonAnywhere (Free Tier)

1. Upload code to PythonAnywhere
2. Configure WSGI file
3. Set up virtual environment
4. Link database (free MySQL)
5. Reload web app

## Frontend Deployment

### Option A: Vercel (Recommended)
```bash
npm install -g vercel
cd frontend
vercel
```

### Option B: Netlify
1. Push to GitHub
2. Connect repo to Netlify
3. Build Command: `npm run build`
4. Publish Directory: `build`

---

## Update API URLs

After deployment, update frontend API base URL:

**frontend/src/services/api.js**
```javascript
const API_BASE_URL = 'https://your-backend-url.railway.app/api';
```

---

## Database Setup on Production

Run migrations on deployed server:
```bash
# Via Railway/Render CLI
railway run python manage.py migrate
render-cli execute "python manage.py migrate"
```

Create admin user on production:
```bash
python manage.py createsuperuser
```
